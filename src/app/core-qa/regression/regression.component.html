REGRESSION IN TEST LEVELS

  • At Test Yantra, Unit testing is considered as one of important aspects where major amount of the defects can be stopped from being introduced into the Test Life cycle or into further stages of testing. This Unit tested build is released to QA for executions in the QA environment.
  • Our QA analyst evaluates the stability of build through Smoke testing and if execution is successful, he passes it for further stages of Functional & Non-Functional Test Evaluations.
  • When it is first build evaluation in Cycle 1 – Functional testing like Integration/ System testing is to be looked upon with at most priority. Defects which are detected during this testing are reported by QA and same are being fixed by Development team.
  • Developer shares Build 2 with defect fixes in Cycle 2 – This is the phase where build has to be evaluated to see if earlier detected defects are no more seen and also fix of these defects by developers have not introduced any new defects into application.
  • Defects found during this phase are marked as ‘Regression’ defects. Meaning some part of code which was working well earlier, is not working anymore.

REGRESSION TEST CYCLES

Retesting on new builds for same test cases can ensure closure of defects.

Until regression defects no more show up in the application, retesting has to be done in multiple times considering Functional & Non – Functional aspects of application.

During this testing, priority of test scenarios / test cases is also identified by below-

  1. Defects Severity
  2. Functional/Business need

Regression Testing can be stopped only when the below conditions are seen

  1. No more reoccurring defects seen in application even after multiple build merges
  2. All the opened defects are closed.

IDENTIFYING TEST SCENARIOS FOR REGRESSION

There is a systematic approach to identify the Regression Scenarios. In the Agile era, it is important to keep a check on the schedule of project and design the Test Scenarios/ Cases.

Ideally design of Test Scenarios which cover end to end cases, edge cases, security and performance cases should be part of Regression. A test bed consisting of all possible scenarios have to be noted down and pick from these the following scenarios –

  1. Critical 25% of scenarios which focus on key functionalities of application- Make them part of Smoke Test Scenarios
  2. 75% of rest of scenarios which focus on remaining features of application- Make them part of System Test cases
  3. Together of (a) & (b) should make Regression suite of application
  4. This Regression suite needs constant updation depending on the functionalities of application
  5. Baseline Regression suite should be taken as Test Scenarios for further application Automation
