user acceptance testing

INTRODUCTION

We at Test Yantra validates the software and ensures the software works seamlessly, meets the customer expectation in terms of behaviour & UI and highlight all kinds of Defects.

UAT is phase where Quality of application is determined before it is Live to actual users. Assessing the application as Users is very important aspect of our testing and this is considered in a very systematic approach.

In this testing process the software system is tested for acceptability & validates the end to end business flow. Here, the actual software users are involved in testing the product to validate the product function, required tasks in real-world scenarios, according to specifications against the requirements.

UAT is performed after completing System Testing and almost all of the major defects have been fixed. UAT will be conducted in the final stage of Software Development Life Cycle (SDLC) prior to system being delivered to a live production environment.

UAT – PHASES, VERIFICATION STEPS & PROS

  • UAT gathers the user requirements in a certifiable way
  • Identifies the issue missed during the System testing or Regression testing
  • It facilitates close collaboration between developers, domain experts, and customers
  • It allows an unambiguous contract between QA, Clients and end customers
  • Reduces the chance of new defects and regressions
  • Provide an overview of the system
  • At most giving an early view of application to customers creates confidence in software release to production successfully

USER ACCEPTANCE TEST TYPES

Following are the types of user acceptance testing that exists in Test Yantra:

  • Alpha and Beta Testing: Alpha testing comes into the picture in the early phase of development and involve internal staffs. It can also be performed by potential user groups, but in the development environment before the product is released to customers. The inputs taken from alpha testers are then used to fix certain bugs to improve the overall quality of the product. Unlike Alpha testing, Beta testing happens in the customer’s environment. It involves extensive testing of the product by a group of customers, who provides the feedback, which is then implemented to improve the product’s quality. Both alpha and beta testing is done before the application is released.
  • Contract Acceptance Testing: Here the software that is developed is tested against certain criteria. The criteria and specifications are defined when the contract is made and agreed upon.
  • Black Box Testing: This kind of user acceptance testing is often categorized as functional testing. It allows the tester to analyse certain functionalities without letting the testers to see the internal code. The principles of black box testing are similar to user acceptance testing, as the user is unaware of the code, but only the requirements that should be met by the software.
