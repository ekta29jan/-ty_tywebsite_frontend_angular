Functional Integration System Testing

FIST Introduction

Functional Testing or FIST is a testing technique used in Software Testing used to test each function of the software application to see if it operates in conformance with the requirement specification. This is a Black Box testing technique and focuses on manual testing as well as automation testing.

FIST is Functional Integration System Testing which forms the main aspects of Functional Test scenarios. Through this type of Testing complete feature functionalities are assessed.

Functional Testing involves checking of the following
  • User Interface
  • API's
  • Database
  • Security
  • Client/ server applications
  • Functionality of the Application Under Test

FIST Testing

At Test Yantra Functional Testing defines the process of testing of reliability, usability etc.

Parameters Functional Testing Non-Functional Testing
Objective It carries out the verified software actions. It is used to verify the performance of the software.

FIST Testing Types

Unit Testing

At Test Yantra Unit Testing is the first phase of validation where we assess the Unit Testcase execution results. Unit Testing is generally done by developers or by the Automation experts.

Unit testing is the first level of testing features of an application.

When Unit testing is 100% correct, 80% of testing is taken care during this phase.

As every testing entry criteria , Unit testing execution reports are a must.

Smoke Testing & Sanity Testing

Smoke Test

Checking the basic & critical feature of an application before processing with deep regressing testing is known as smoke testing.

Sanity Test

This is a testing type referred to check added features , also sanity test ensures testing bed is ready to test all features of application.

Integration Testing

Integration Testing ensures flow of data from one module to another module, making flows of application functionally right.

Types of Integration:

  • Incremental testing
    • Top down Approach
  • Non – Incremental testing or Big Bang theory
    • Bottom up Approach
Incremental Integration testing

We go for this approach whenever there is a clear relationship between modules.

Ex: Booking Ticket, shopping

Non Incremental Integration testing

We go for this approach whenever there is no clear relationship between the modules.

EX: we create data in one modules & check for the saved data by banking on all other.

Before integration testing each & every module working independently.

System Testing

This test type is one of important aspects of Software Testing, ensuring end to end features of application are working fine.

Here a user navigates through all the require modules of an application and check the system as a whole (Respective to their Assign Modules).

This Testing ensures end to end flow of a feature or a requirement.

Ex :- User logging into Gmail , checking inbox and logging out

Types of System Test

System Testing is usually carried out at Test yantra by a team that is independent of the development team in order to measure the quality of the system unbiased. It includes both functional and Non-Functional testing.

System Testing Process

At Test Yantra every build has to get assessed for System Testing, which is performed step by step in the following ways:

  • Setting up Right Environment: Create testing environment for the better quality testing.
  • Prepare Test Case: Generate test case for the testing process.
  • Analyze and prepare required Test Data: Generate the data that is to be tested.
  • Test Case Execution: After the generation of the test case and the test data, test cases are executed.
  • Defect Identification & Reporting: Defects in the system are detected.
  • Regression Test:It is carried out to test the side effects of the testing process.
  • Log Defects: Defects are fixed in this step.
  • Re-execute & Retest: If the test is not successful then again test is performed.

Advantages of System Testing

  • Detects the maximum bugs
  • Verifies system against all kinds of requirements
  • Creates confidence among the team for release of software
  • Uses the test environment similar to real/ business environment
  • Testers do not need coding knowledge to test this as its black box testing
